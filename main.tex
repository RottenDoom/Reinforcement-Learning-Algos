\begin{abstract}
This report presents a comparative study of nine model-free reinforcement learning
(RL) algorithms implemented and evaluated on an $8\times8$ stochastic
Gridworld cost-minimisation task.  The algorithms span four families: off-policy
TD control (Q-learning, Double Q-learning, Entropy-Regularised Q-learning /
G-learning), on-policy TD control (SARSA, Expected SARSA), Monte-Carlo policy
gradient (REINFORCE, REINFORCE with baseline), and online actor-critic
(one-step AC, AC($\lambda$)).  All algorithms were trained for 1500 episodes
across 10 independent random seeds under identical environment instances,
discount factor ($\gamma = 0.85$), and a shared count-based learning-rate
schedule ($\alpha_t = n_t(s,a)^{-0.8}$).  Algorithm-specific hyperparameters
were selected via coordinate-descent sweeps prior to the final comparison.
Performance was measured via three metrics: signed relative error, absolute
relative error, and policy evaluation error.  Results show that Q-learning
achieves the lowest mean final policy evaluation error ($0.076 \pm 0.029$),
followed closely by G-learning ($0.094 \pm 0.028$).  Pure REINFORCE produces
the largest error ($0.576 \pm 0.096$) due to high-variance Monte-Carlo returns.
A notable tuning finding is that eligibility traces ($\lambda>0$) actively hurt
actor-critic performance in this wall-heavy environment, making AC($\lambda$)
with $\lambda=0$ equivalent to one-step AC.  The findings confirm classical RL
theory: off-policy value-based methods with adaptive learning rates outperform
policy-gradient approaches on tabular, cost-minimisation MDPs where Bellman
signals are available at every step.
\end{abstract}

% ============================================================
\section{Introduction}
% ============================================================

Reinforcement learning (RL) is a sequential decision-making framework in which
an agent learns a behaviour policy by interacting with an environment, receiving
reward (or cost) signals, without being given a labelled dataset or an
explicit model of the world.  Its uniqueness lies in the \emph{credit-assignment
problem}: the agent must discover, purely from experience, which actions taken
at which states are responsible for eventual outcomes that may be separated from
the deciding moment by many time steps.

RL is the natural tool for a problem when three attributes are simultaneously
present.  First, the task is \emph{sequential}: decisions affect future states
and thus future decisions, so a greedy myopic policy is sub-optimal.  Second,
the \emph{environment model is unavailable or too complex to use}: transition
probabilities and cost functions are either unknown or have too many degrees
of freedom for explicit dynamic programming.  Third, there is a well-defined
\emph{scalar performance signal} that can be computed after each action.
Classic examples of use-cases in real life include robotic locomotion,
game-playing, inventory management, and the navigation task studied in this
project, where an agent must reach a goal in a stochastic grid while minimising
cumulative movement cost and avoiding walls.

% ============================================================
\section{Problem Setup}
% ============================================================

\subsection{Gridworld Environment}

The environment is the $8 \times 8$ stochastic Gridworld introduced by Fox
et~al.~(2016).  States are cells of the grid, indexed by a flat integer
$s = 8r + c$ where $r \in \{0,\ldots,7\}$ is the row and $c \in \{0,\ldots,7\}$
is the column.

\paragraph{State space.}
Of the 64 raw grid cells, 15 are permanently blocked (walls) and therefore
unreachable.  The blocked flat indices are $\{9, 12, 17, 20, 25, 28, 29, 30,
33, 34, 38, 42, 46, 50, 54\}$.  The remaining 48 non-terminal, non-blocked
cells form the effective state space.

\paragraph{Action space.}
Nine deterministic intended-move actions: N, NE, E, SE, S, SW, W, NW, and
Stay (action indices 0--8).  The \emph{intended} move is always attempted
first; the actual outcome is stochastic due to drift (see below).

\paragraph{Transition kernel.}
After the intended move resolves, a stochastic drift is applied independently.
With probability 0.70 no drift occurs; with probability 0.05 each the agent
drifts one step in each cardinal direction; and with probability 0.025 each
it drifts diagonally.  If the intended move or drift would take the agent
out-of-bounds or into a blocked cell, the move is \emph{reverted} to the
pre-move position.

\paragraph{Cost structure.}
Each step incurs a base cost of $1 + \mathcal{N}(0, 0.04)$ (standard
deviation 0.2).  If the intended move is blocked by a wall, an additional
penalty of $1000 + \mathcal{N}(0, 0.04)$ is added, creating a steep incentive
to avoid walls.  Drift that results in an actual displacement from the
post-action position incurs an extra cost of $1 + \mathcal{N}(0, 0.04)$.

\paragraph{Terminal state.}
Flat index 36 (row~4, column~4) is the goal.  It is absorbing: once reached,
the episode ends with zero additional cost.

\paragraph{Discount factor.}
$\gamma = 0.85$.  This moderately discounts future costs, balancing short-term
efficiency against long-term path quality.

\subsection{Objective}

The control objective is to find a stationary policy
$\pi^*: \mathcal{S} \to \mathcal{A}$ that minimises the expected discounted
cumulative cost from every starting state:
\begin{equation}
    V^{\pi^*}(s) = \min_{\pi}\; \mathbb{E}_{\pi}\!\left[
        \sum_{t=0}^{\infty} \gamma^t\, c_t \;\Big|\; s_0 = s
    \right], \quad \forall\, s \in \mathcal{S}.
\end{equation}
The known optimal values $V^*$ (computed offline via value iteration on the
model matrices $P$ and $C$) serve as ground truth for all evaluation metrics.
Values range from $V^*(s)\approx 1.0$ for states adjacent to the goal up to
$V^*(s)\approx 5.4$ for far corners.

% ============================================================
\section{Algorithms Implemented}
% ============================================================

All nine algorithms share a common interface: \texttt{select\_action(s)},
\texttt{update(s,\,a,\,c,\,s')}, \texttt{get\_value\_estimate()} and
\texttt{get\_policy()}.  State is always the flat integer index.  The
count-based learning rate $\alpha_t = n_t(s,a)^{-\omega}$ (with $\omega=0.8$,
counts reset each episode and initialised to 1) is shared across all TD
algorithms; policy-gradient methods use a fixed learning rate instead.

% ------------------------------------------------------------
\subsection{Q-Learning}
% ------------------------------------------------------------

Q-learning is an off-policy TD method.  At each step it updates:
\begin{equation}
    Q(s,a) \;\leftarrow\; Q(s,a) + \alpha\bigl[c + \gamma\min_{a'}Q(s',a') - Q(s,a)\bigr].
\end{equation}
The greedy bootstrap $\min_{a'}Q(s',a')$ decouples the target from the
behaviour policy, allowing convergence to $Q^*$ under mild conditions.
Exploration uses $\varepsilon$-greedy with $\varepsilon_0=0.2$, decaying
multiplicatively by $0.9$ per step within each episode.

\begin{algorithm}[H]
\caption{Q-Learning}
\label{alg:qlearning}
\begin{algorithmic}[1]
\State \textbf{Initialise} $Q(s,a)\leftarrow 0$,\; $n(s,a)\leftarrow 1$ \quad for all $s,a$
\For{each episode $e = 1,\ldots,N$}
    \State $s \leftarrow \texttt{env.reset()}$;\; $\varepsilon \leftarrow \varepsilon_0$
    \For{each step $t = 1,\ldots,T_{\max}$}
        \State $a \;\leftarrow\; \varepsilon\text{-greedy}\bigl(Q(s,\cdot)\bigr)$
        \State $(s',\, c,\, \textit{done}) \leftarrow \texttt{env.step}(a)$
        \State $\alpha \leftarrow n(s,a)^{-\omega}$
        \State $Q(s,a) \;\leftarrow\; Q(s,a) + \alpha\bigl[c + \gamma\min_{a'}Q(s',a') - Q(s,a)\bigr]$
        \State $n(s,a)\leftarrow n(s,a)+1$;\; $\varepsilon\leftarrow\varepsilon\cdot\varepsilon_{\text{decay}}$;\; $s\leftarrow s'$
        \If{\textit{done}} \textbf{break} \EndIf
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

\textbf{Hyperparameter tuning.}  $\varepsilon_0=0.2$ was selected via sweep
(Section~\ref{sec:tuning}): lower initial randomness reduces early wall-collision
penalties while still covering the state space within each 300-step episode.
The decay factor 0.9 ensures $\varepsilon$ collapses to ${\approx}0.001$ after
50 steps, promoting exploitation for the remaining 250 steps of each episode.

% ------------------------------------------------------------
\subsection{Double Q-Learning}
% ------------------------------------------------------------

Double Q-learning maintains two tables $Q_A$ and $Q_B$.
At each step, one table is chosen with probability 0.5 to be updated; the
other evaluates the target:
\begin{equation}
    \text{target} = c + \gamma\, Q_B\!\bigl(s',\, \arg\min_{a'} Q_A(s',a')\bigr).
\end{equation}
This decouples \emph{action selection} from \emph{action evaluation},
reducing the maximisation bias present in standard Q-learning.  Exploration
uses the mean $(Q_A+Q_B)/2$.

\begin{algorithm}[H]
\caption{Double Q-Learning}
\label{alg:doubleq}
\begin{algorithmic}[1]
\State \textbf{Initialise} $Q_A(s,a)\leftarrow 0$, $Q_B(s,a)\leftarrow 0$,\;
       $n_A(s,a)\leftarrow 1$, $n_B(s,a)\leftarrow 1$
\For{each episode}
    \State $s \leftarrow \texttt{env.reset()}$;\; $\varepsilon \leftarrow \varepsilon_0$
    \For{each step}
        \State $a \leftarrow \varepsilon\text{-greedy}\!\bigl((Q_A+Q_B)/2,\;\text{at }s\bigr)$
        \State $(s',\, c,\, \textit{done}) \leftarrow \texttt{env.step}(a)$
        \If{$\text{Uniform}(0,1) < 0.5$} \Comment{update table $A$}
            \State $a^* \leftarrow \arg\min_{a'} Q_A(s',a')$
            \State $Q_A(s,a) \leftarrow Q_A(s,a) + \alpha_A\bigl[c + \gamma Q_B(s',a^*) - Q_A(s,a)\bigr]$
            \State $n_A(s,a)\leftarrow n_A(s,a)+1$;\; $\alpha_A \leftarrow n_A(s,a)^{-\omega}$
        \Else \Comment{update table $B$}
            \State $a^* \leftarrow \arg\min_{a'} Q_B(s',a')$
            \State $Q_B(s,a) \leftarrow Q_B(s,a) + \alpha_B\bigl[c + \gamma Q_A(s',a^*) - Q_B(s,a)\bigr]$
            \State $n_B(s,a)\leftarrow n_B(s,a)+1$;\; $\alpha_B \leftarrow n_B(s,a)^{-\omega}$
        \EndIf
        \State $\varepsilon\leftarrow\varepsilon\cdot\varepsilon_{\text{decay}}$;\; $s\leftarrow s'$
        \If{\textit{done}} \textbf{break} \EndIf
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

\textbf{Hyperparameter tuning.}  Separate visit counts per table are maintained
so each table's learning rate adapts only to how often it was updated,
preventing one table from ageing prematurely.  Shares the same
$\varepsilon_0=0.2$ schedule as Q-learning.

% ------------------------------------------------------------
\subsection{SARSA}
% ------------------------------------------------------------

SARSA is an on-policy TD method.  The next action
$a'$ is sampled from the current $\varepsilon$-greedy policy \emph{before}
the update:
\begin{equation}
    Q(s,a) \;\leftarrow\; Q(s,a) + \alpha\bigl[c + \gamma\, Q(s',a') - Q(s,a)\bigr].
\end{equation}
Because the target uses the same policy that generates behaviour, SARSA
optimises for the \emph{behaviour} policy and is inherently cautious near
dangerous states (walls).

\begin{algorithm}[H]
\caption{SARSA}
\label{alg:sarsa}
\begin{algorithmic}[1]
\State \textbf{Initialise} $Q(s,a)\leftarrow 0$,\; $n(s,a)\leftarrow 1$
\For{each episode}
    \State $s \leftarrow \texttt{env.reset()}$;\; $\varepsilon \leftarrow \varepsilon_0$
    \State $a \leftarrow \varepsilon\text{-greedy}(Q(s,\cdot))$ \Comment{select first action on-policy}
    \For{each step}
        \State $(s',\, c,\, \textit{done}) \leftarrow \texttt{env.step}(a)$
        \State $a' \leftarrow \varepsilon\text{-greedy}(Q(s',\cdot))$ \Comment{on-policy next action}
        \State $\alpha \leftarrow n(s,a)^{-\omega}$
        \State $Q(s,a) \leftarrow Q(s,a) + \alpha\bigl[c + \gamma Q(s',a') - Q(s,a)\bigr]$
        \State $n(s,a)\leftarrow n(s,a)+1$;\; $\varepsilon\leftarrow\varepsilon\cdot\varepsilon_{\text{decay}}$
        \State $s\leftarrow s'$;\; $a\leftarrow a'$
        \If{\textit{done}} \textbf{break} \EndIf
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

\textbf{Hyperparameter tuning.}  Same $\varepsilon_0=0.2$ schedule as Q-learning.
The on-policy nature means no additional tuning is needed; the conservative
tendency is an emergent structural property.

% ------------------------------------------------------------
\subsection{Expected SARSA}
% ------------------------------------------------------------

Expected SARSA replaces the sampled $Q(s', a')$ with its expectation under
the current $\varepsilon$-greedy policy:
\begin{equation}
    \text{target} = c + \gamma \sum_{a'} \pi(a'|s')\, Q(s',a').
\end{equation}
This eliminates one source of variance compared to SARSA (no random sample
of $a'$) while remaining on-policy.  In the limit $\varepsilon \to 0$ it
reduces to Q-learning.

\begin{algorithm}[H]
\caption{Expected SARSA}
\label{alg:esarsa}
\begin{algorithmic}[1]
\State \textbf{Initialise} $Q(s,a)\leftarrow 0$,\; $n(s,a)\leftarrow 1$
\For{each episode}
    \State $s \leftarrow \texttt{env.reset()}$;\; $\varepsilon \leftarrow \varepsilon_0$
    \For{each step}
        \State $a \leftarrow \varepsilon\text{-greedy}(Q(s,\cdot))$
        \State $(s',\, c,\, \textit{done}) \leftarrow \texttt{env.step}(a)$
        \State $\pi(a'|s') \leftarrow \tfrac{\varepsilon}{|\mathcal{A}|} + (1-\varepsilon)\cdot\mathbf{1}[a'=\arg\min_a Q(s',a)]$
        \State $\bar{V}(s') \leftarrow \sum_{a'}\pi(a'|s')\,Q(s',a')$ \Comment{expected value, no sampling}
        \State $\alpha \leftarrow n(s,a)^{-\omega}$
        \State $Q(s,a) \leftarrow Q(s,a) + \alpha\bigl[c + \gamma\bar{V}(s') - Q(s,a)\bigr]$
        \State $n(s,a)\leftarrow n(s,a)+1$;\; $\varepsilon\leftarrow\varepsilon\cdot\varepsilon_{\text{decay}}$;\; $s\leftarrow s'$
        \If{\textit{done}} \textbf{break} \EndIf
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

\textbf{Hyperparameter tuning.}  Identical schedule to SARSA; the variance
reduction is structural.

% ------------------------------------------------------------
\subsection{Entropy-Regularised Q-Learning (G-Learning)}
% ------------------------------------------------------------

G-learning augments the Bellman equation with a KL
divergence penalty relative to a uniform prior $\rho(a|s)=1/9$, replacing
the hard-$\min$ bootstrap with a softmin value:
\begin{equation}
    V(s') = -\frac{1}{\beta}\log\!\left(\sum_{a'} e^{-\beta\, G(s',a')}\right)
    \quad(\text{with min-subtraction for numerical stability}).
\end{equation}
The induced policy is $\mu(a|s) \propto \exp(-\beta\, G(s,a))$.  The inverse
temperature $\beta$ follows a linear schedule:
\begin{equation}
    \beta_t = \beta_{\min} + t\,\frac{\beta_{\max}-\beta_{\min}}{N},\quad
    \beta_{\min}=0.01,\quad \beta_{\max}=7.0.
\end{equation}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.6\linewidth]{../report/figures/01_beta_schedule.png}
    \caption{Linear $\beta$ annealing schedule for G-learning.  At low $\beta$
    the softmin policy is nearly uniform (maximum exploration); as $\beta$
    increases the policy sharpens toward greedy.}
    \label{fig:beta_schedule}
\end{figure}

\begin{algorithm}[H]
\caption{G-Learning (Entropy-Regularised Q-Learning)}
\label{alg:glearning}
\begin{algorithmic}[1]
\State \textbf{Initialise} $G(s,a)\leftarrow 0$,\; $n(s,a)\leftarrow 1$
\For{each episode $e = 1,\ldots,N$}
    \State $\beta \leftarrow \beta_{\min} + (e/N)(\beta_{\max}-\beta_{\min})$ \Comment{anneal temperature}
    \State $s \leftarrow \texttt{env.reset()}$;\; $\varepsilon \leftarrow \varepsilon_0$
    \For{each step}
        \State $\mu(a|s) \propto \exp\!\bigl(-\beta\,G(s,a)\bigr)$ \Comment{softmin policy}
        \State $a \leftarrow \varepsilon\text{-greedy}(\mu(\cdot|s))$
        \State $(s',\, c,\, \textit{done}) \leftarrow \texttt{env.step}(a)$
        \State $G_{\min} \leftarrow \min_{a'} G(s',a')$ \Comment{subtract for stability}
        \State $V(s') \leftarrow -\tfrac{1}{\beta}\log\!\bigl(\textstyle\sum_{a'}\exp(-\beta(G(s',a')-G_{\min}))\bigr) - G_{\min}$
        \State $\alpha \leftarrow n(s,a)^{-\omega}$
        \State $G(s,a) \leftarrow G(s,a) + \alpha\bigl[c + \gamma V(s') - G(s,a)\bigr]$
        \State $n(s,a)\leftarrow n(s,a)+1$;\; $\varepsilon\leftarrow\varepsilon\cdot\varepsilon_{\text{decay}}$;\; $s\leftarrow s'$
        \If{\textit{done}} \textbf{break} \EndIf
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

\textbf{Hyperparameter tuning.}  $\beta_{\min}=0.01$ (tuned from 0.1) produces
a near-uniform softmin policy in early episodes, maximising exploration of the
wall-heavy grid.  $\beta_{\max}=7.0$ is confirmed as the best upper limit,
approximating greedy behaviour at convergence without sharpening prematurely.

% ------------------------------------------------------------
\subsection{REINFORCE}
% ------------------------------------------------------------

REINFORCE is a Monte-Carlo policy gradient method.  A full episode is collected,
the discounted return $G_t = \sum_{k=t}^{T}\gamma^{k-t}c_k$
is computed at each step, and a softmax policy
$\pi(a|s)=\operatorname{softmax}(\theta_s)$ is updated by gradient descent on
the expected cost:
\begin{equation}
    \theta_s \;\leftarrow\; \theta_s + \alpha\, G_t\,\bigl(\pi(\cdot|s) - \mathbf{e}_{a_t}\bigr).
\end{equation}

\begin{algorithm}[H]
\caption{REINFORCE}
\label{alg:reinforce}
\begin{algorithmic}[1]
\State \textbf{Initialise} $\theta(s,a)\leftarrow 0$ for all $s,a$
\For{each episode}
    \State Collect trajectory $\tau = \{(s_0,a_0,c_0),\ldots,(s_T,a_T,c_T)\}$
           by sampling $a_t \sim \pi(\cdot|s_t) = \operatorname{softmax}(\theta(s_t,\cdot))$
    \For{$t = 0,\ldots,T$}
        \State $G_t \leftarrow \sum_{k=t}^{T} \gamma^{k-t}\, c_k$
        \State $\theta(s_t,a_t) \leftarrow \theta(s_t,a_t) + \alpha\cdot G_t\cdot\bigl(1-\pi(a_t|s_t)\bigr)$
        \State $\theta(s_t,a)\;\; \leftarrow \theta(s_t,a)\;\; - \alpha\cdot G_t\cdot\pi(a|s_t)$
               \hfill for all $a\neq a_t$
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

\textbf{Hyperparameter tuning.}  Fixed $\alpha=0.001$ (tuned from 0.01).  The
wall-collision penalty of ${\approx}1001$ dominates individual returns, producing
extremely large gradient magnitudes; a ten-fold smaller learning rate is
necessary to prevent policy oscillation across seeds.

% ------------------------------------------------------------
\subsection{REINFORCE with Baseline}
% ------------------------------------------------------------

A learned state-value baseline $V(s)$ is subtracted from $G_t$ to reduce
gradient variance:
\begin{align}
    \delta_t &= G_t - V(s_t),\\
    V(s_t) &\;\leftarrow\; V(s_t) + \alpha_b\,\delta_t,\\
    \theta_s &\;\leftarrow\; \theta_s + \alpha\,\delta_t\,\bigl(\pi(\cdot|s) - \mathbf{e}_{a_t}\bigr).
\end{align}
The baseline does not introduce bias because $V(s_t)$ is independent of the
action taken.

\begin{algorithm}[H]
\caption{REINFORCE with Baseline}
\label{alg:reinforce_baseline}
\begin{algorithmic}[1]
\State \textbf{Initialise} $\theta(s,a)\leftarrow 0$,\; $V(s)\leftarrow 0$
\For{each episode}
    \State Collect trajectory $\tau = \{(s_t, a_t, c_t)\}$ following
           $\pi(\cdot|s) = \operatorname{softmax}(\theta(s,\cdot))$
    \For{$t = 0,\ldots,T$}
        \State $G_t \leftarrow \sum_{k=t}^{T} \gamma^{k-t}\, c_k$
        \State $\delta_t \leftarrow G_t - V(s_t)$ \Comment{centred advantage}
        \State $V(s_t) \leftarrow V(s_t) + \alpha_b\cdot\delta_t$
        \State $\theta(s_t,a_t) \leftarrow \theta(s_t,a_t) + \alpha\cdot\delta_t\cdot\bigl(1-\pi(a_t|s_t)\bigr)$
        \State $\theta(s_t,a)\;\; \leftarrow \theta(s_t,a)\;\; - \alpha\cdot\delta_t\cdot\pi(a|s_t)$
               \hfill for all $a\neq a_t$
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.75\linewidth]{../report/figures/03_variance_comparison.png}
    \caption{Return variance comparison between REINFORCE and REINFORCE with
    baseline across training.  The baseline rapidly reduces effective gradient
    variance once the value function $V(s)$ has learned a reasonable estimate
    of $V^\pi$.}
    \label{fig:variance_comparison}
\end{figure}

\textbf{Hyperparameter tuning.}  $\alpha_b = 0.05$ (baseline), $\alpha = 0.001$
(policy).  A moderately smaller baseline LR than the default (0.1) prevents the
value function from overshooting during the early high-variance episodes caused
by wall collisions.

% ------------------------------------------------------------
\subsection{One-Step Actor-Critic}
% ------------------------------------------------------------

The one-step actor-critic updates online at every step using the TD error
$\delta = c + \gamma V(s') - V(s)$ as the learning signal for both critic
and actor:
\begin{align}
    V(s) &\;\leftarrow\; V(s) + \alpha_c\,\delta,\\
    \theta_s &\;\leftarrow\; \theta_s + \alpha_a\,\delta\,\bigl(\pi(\cdot|s) - \mathbf{e}_{a_t}\bigr).
\end{align}

\begin{algorithm}[H]
\caption{One-Step Actor-Critic}
\label{alg:ac_onestep}
\begin{algorithmic}[1]
\State \textbf{Initialise} $\theta(s,a)\leftarrow 0$,\; $V(s)\leftarrow 0$
\For{each episode}
    \State $s \leftarrow \texttt{env.reset()}$
    \For{each step}
        \State $a \sim \pi(\cdot|s) = \operatorname{softmax}(\theta(s,\cdot))$
        \State $(s',\, c,\, \textit{done}) \leftarrow \texttt{env.step}(a)$
        \State $\delta \leftarrow c + \gamma V(s') - V(s)$ \Comment{one-step TD error}
        \State $V(s) \leftarrow V(s) + \alpha_c\cdot\delta$ \Comment{critic update}
        \State $\theta(s,a)\;\; \leftarrow \theta(s,a)\;\; + \alpha_a\cdot\delta\cdot\bigl(1-\pi(a|s)\bigr)$
        \State $\theta(s,a') \leftarrow \theta(s,a') - \alpha_a\cdot\delta\cdot\pi(a'|s)$
               \hfill for all $a'\neq a$ \Comment{actor update}
        \State $s \leftarrow s'$
        \If{\textit{done}} \textbf{break} \EndIf
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

\textbf{Hyperparameter tuning.}  $\alpha_c = \alpha_a = 0.005$ (tuned from 0.01).
Online updates reduce per-episode variance compared to REINFORCE at the cost of
some bias; the tuned LR prevents actor oscillation from large early TD errors.

% ------------------------------------------------------------
\subsection{Actor-Critic with Eligibility Traces (AC($\lambda$))}
% ------------------------------------------------------------

AC($\lambda$) maintains eligibility trace vectors $\mathbf{z}_V$ (shape 64)
and $\mathbf{z}_\theta$ (shape $64\times9$).  At each step:
\begin{align}
    \mathbf{z}_V &\;\leftarrow\; \gamma\lambda_c\,\mathbf{z}_V + \mathbf{e}_s,\\
    \mathbf{z}_\theta &\;\leftarrow\; \gamma\lambda_a\,\mathbf{z}_\theta
        + \bigl(\pi(\cdot|s)-\mathbf{e}_{a_t}\bigr)\,\text{at row }s,\\
    V &\;\leftarrow\; V + \alpha_c\,\delta\,\mathbf{z}_V,\qquad
    \theta \;\leftarrow\; \theta + \alpha_a\,\delta\,\mathbf{z}_\theta.
\end{align}
Traces are reset to zero at the start of each episode.

\begin{algorithm}[H]
\caption{Actor-Critic with Eligibility Traces AC($\lambda$)}
\label{alg:ac_lambda}
\begin{algorithmic}[1]
\State \textbf{Initialise} $\theta(s,a)\leftarrow 0$,\; $V(s)\leftarrow 0$
\For{each episode}
    \State $\mathbf{z}_V \leftarrow \mathbf{0}_{64}$;\;
           $\mathbf{z}_\theta \leftarrow \mathbf{0}_{64\times 9}$ \Comment{reset traces}
    \State $s \leftarrow \texttt{env.reset()}$
    \For{each step}
        \State $a \sim \pi(\cdot|s) = \operatorname{softmax}(\theta(s,\cdot))$
        \State $(s',\, c,\, \textit{done}) \leftarrow \texttt{env.step}(a)$
        \State $\delta \leftarrow c + \gamma V(s') - V(s)$
        \State $\mathbf{z}_V \leftarrow \gamma\lambda_c\,\mathbf{z}_V$;\;
               $z_V[s] \leftarrow z_V[s] + 1$ \Comment{accumulating trace}
        \State $\mathbf{z}_\theta \leftarrow \gamma\lambda_a\,\mathbf{z}_\theta$
        \State $z_\theta[s,a]\;\; \leftarrow z_\theta[s,a]\;\; + \bigl(1-\pi(a|s)\bigr)$
        \State $z_\theta[s,a'] \leftarrow z_\theta[s,a'] - \pi(a'|s)$
               \hfill for all $a'\neq a$
        \State $V \leftarrow V + \alpha_c\cdot\delta\cdot\mathbf{z}_V$ \Comment{global update via traces}
        \State $\theta \leftarrow \theta + \alpha_a\cdot\delta\cdot\mathbf{z}_\theta$
        \State $s \leftarrow s'$
        \If{\textit{done}} \textbf{break} \EndIf
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

\textbf{Hyperparameter tuning.}  $\lambda_c = \lambda_a = 0$ (tuned from 0.9).
The $\lambda$ sweep (Section~\ref{sec:tuning}) shows that non-zero traces
increase policy error monotonically on this environment.  The 1000-unit
wall-collision penalty creates extremely large TD errors; when propagated
backward through non-zero traces these corrupt value and policy estimates at
many unrelated states.  At $\lambda=0$, AC($\lambda$) reduces exactly to
one-step AC.

\section{Advanced Policy Optimisation Algorithms}
\label{sec:advanced}
% ============================================================

Notebooks 06--09 implement four additional algorithms that differ from the nine
core methods primarily in \emph{how} the policy gradient is computed or applied,
rather than in the RL formulation.  All four are episodic, share the same
tabular softmax policy parameterisation, and are integrated into the same
\texttt{run\_all.py} pipeline.

% ------------------------------------------------------------
\subsection{SGD Variants (Vanilla, Momentum, Nesterov)}
% ------------------------------------------------------------

All three variants use the same REINFORCE gradient estimate.  After collecting
an episode the per-state gradient is:
\[
    g[s,:] = \sum_{t:\,s_t=s} G_t\,(\pi(\cdot|s) - \mathbf{e}_{a_t}).
\]
They differ only in the optimiser applied to $\theta$:

\begin{algorithm}[H]
\caption{SGD Policy Gradient (vanilla / momentum / Nesterov)}
\label{alg:sgd}
\begin{algorithmic}[1]
\State \textbf{Initialise} $\theta \leftarrow \mathbf{0}$;\;
       $\mathbf{v} \leftarrow \mathbf{0}$ (momentum buffer)
\For{each episode}
    \State Collect $\tau = \{(s_t,a_t,c_t)\}$;\; compute $G_t$ for each step
    \State Accumulate $g[s,:] \leftarrow \sum_t G_t(\pi(\cdot|s)-\mathbf{e}_{a_t})$ per state
    \For{each visited state $s$}
        \If{$\beta = 0$} \Comment{vanilla SGD}
            \State $\theta[s,:] \;\leftarrow\; \theta[s,:] + \eta\, g[s,:]$
        \ElsIf{not Nesterov} \Comment{heavy-ball}
            \State $\mathbf{v}[s,:] \leftarrow \beta\,\mathbf{v}[s,:] + g[s,:]$
            \State $\theta[s,:] \leftarrow \theta[s,:] + \eta\,\mathbf{v}[s,:]$
        \Else \Comment{Nesterov}
            \State $\mathbf{v}_{\text{prev}} \leftarrow \mathbf{v}[s,:]$;\;
                   $\mathbf{v}[s,:] \leftarrow \beta\,\mathbf{v}[s,:] + g[s,:]$
            \State $\theta[s,:] \leftarrow \theta[s,:] + \eta\,(\beta\,\mathbf{v}[s,:] + g[s,:] - \mathbf{v}_{\text{prev}})$
        \EndIf
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

\textbf{Hyperparameters.}  Vanilla SGD: $\eta=0.005$.  Heavy-ball: $\eta=0.001$,
$\beta=0.9$.  Nesterov: $\eta=0.001$, $\beta=0.99$.

% ------------------------------------------------------------
\subsection{Policy Mirror Descent}
% ------------------------------------------------------------

Mirror Descent replaces the Euclidean gradient step with a multiplicative
update derived from the KL-divergence Bregman potential.  For a softmax policy
this corresponds to a direct scaling of action probabilities:
\[
    \pi_{\text{new}}(a|s) \;\propto\; \pi_{\text{old}}(a|s)\cdot
    \exp\!\bigl(-\eta\, G_t\cdot\mathbf{1}[a = a_t]\bigr).
\]
In log-probability space this is an additive shift followed by renormalisation,
keeping the policy on the simplex by construction.

\begin{algorithm}[H]
\caption{Policy Mirror Descent}
\label{alg:md}
\begin{algorithmic}[1]
\State \textbf{Initialise} $\log\pi[s,a] \leftarrow -\log|\mathcal{A}|$ (uniform)
\For{each episode}
    \State Collect $\tau = \{(s_t,a_t,c_t)\}$;\; compute $G_t$
    \For{$t = T,\ldots,0$} \Comment{backward pass}
        \State $\log\pi[s_t,a_t] \;\leftarrow\; \log\pi[s_t,a_t] - \eta\,G_t$
        \State $\log\pi[s_t,:] \;\leftarrow\; \log\pi[s_t,:] - \log Z_{s_t}$
               \Comment{renormalise}
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

\textbf{Hyperparameters.}  Step size $\eta=0.005$ (tuned, notebook 07).

% ------------------------------------------------------------
\subsection{Natural Policy Gradient}
% ------------------------------------------------------------

The Natural Policy Gradient (NPG) pre-conditions the gradient by the inverse
Fisher information matrix $F(\theta)^{-1}$.  For tabular softmax this has the
closed-form simplification that the natural gradient at state $s$ equals the
advantage function—no matrix inversion is required:
\[
    F_s^{-1}\,g_s \;=\; A(s,\cdot) \;=\; \hat{Q}(s,\cdot) - \hat{V}(s),
\]
where $\hat{Q}(s,a)$ is a running first-visit Monte-Carlo mean and
$\hat{V}(s) = \mathbb{E}_{\pi}[\hat{Q}(s,\cdot)]$.

\begin{algorithm}[H]
\caption{Natural Policy Gradient (tabular)}
\label{alg:npg}
\begin{algorithmic}[1]
\State \textbf{Initialise} $\theta\leftarrow\mathbf{0}$;\;
       $\hat{Q}(s,a)\leftarrow 0$;\; $n(s,a)\leftarrow 0$
\For{each episode}
    \State Collect $\tau$;\; compute first-visit returns $G_t$
    \For{each unique $(s,a)$ in $\tau$}
        \State $n(s,a)\leftarrow n(s,a)+1$;\;
               $\hat{Q}(s,a) \leftarrow \hat{Q}(s,a) + \tfrac{1}{n}(G_t - \hat{Q}(s,a))$
    \EndFor
    \For{each visited state $s$}
        \State $\hat{V}(s) \leftarrow \sum_a \pi(a|s)\,\hat{Q}(s,a)$
        \State $\theta[s,:] \leftarrow \theta[s,:] + \eta\,(\hat{Q}(s,:) - \hat{V}(s))$
               \Comment{natural gradient = advantage}
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

\textbf{Hyperparameters.}  $\eta=0.005$ (tuned, notebook 08).

% ------------------------------------------------------------
\subsection{Trust Region Policy Optimisation (TRPO)}
% ------------------------------------------------------------

TRPO constrains each update to satisfy a KL-divergence trust region:
\[
    \bar{\text{KL}}(\pi_{\text{old}},\pi_{\text{new}}) \;\leq\; \delta.
\]
The optimal step size from the quadratic approximation is
$\alpha_{\max} = \sqrt{2\delta / \sum_s w_s\,A_s^\top F_s A_s}$,
followed by a backtracking line search that halves $\alpha$ until the exact
constraint is satisfied.

\begin{algorithm}[H]
\caption{TRPO (tabular)}
\label{alg:trpo}
\begin{algorithmic}[1]
\State \textbf{Initialise} $\theta\leftarrow\mathbf{0}$;\; $\hat{Q}\leftarrow 0$
\For{each episode}
    \State Collect $\tau$;\; update $\hat{Q}$ (running MC mean, same as NPG)
    \State Compute direction $d[s,:] \leftarrow \hat{Q}(s,:) - \hat{V}(s)$
    \State $\alpha \leftarrow \min\!\bigl(\alpha_{\max},\;\sqrt{2\delta / \sum_s w_s\,d_s^\top F_s d_s}\bigr)$
    \For{$k = 1,\ldots,K$} \Comment{backtracking line search}
        \State $\theta' \leftarrow \theta$;\;
               $\theta'[s,:] \leftarrow \theta[s,:] + \alpha\,d[s,:]$ for all $s$
        \If{$\overline{\text{KL}}(\theta, \theta') \leq \delta$}
            \State $\theta \leftarrow \theta'$;\; \textbf{break}
        \EndIf
        \State $\alpha \leftarrow \alpha / 2$
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}

\textbf{Hyperparameters.}  $\delta=0.01$, $K=5$, $\alpha_{\max}=1.0$
(tuned, notebook 09).  The $\delta$ sweep found the performance surface flat
across all tested values (policy error $\approx 1.39$--$1.41$).

\subsection{Performance Metrics}

Three metrics are recorded after each episode using the agent's current
value estimate $\hat{V}(s)$ and induced policy $\hat{\pi}$:

\begin{enumerate}[leftmargin=*]
    \item \textbf{Signed relative error} (value bias):
        \[
            e_\pm = \frac{1}{|\mathcal{S}_v|}
            \sum_{s\in\mathcal{S}_v} \frac{\hat{V}(s) - V^*(s)}{V^*(s)},
        \]
        where $\mathcal{S}_v$ excludes blocked and terminal states.
        Positive values indicate over-estimation (optimism); negative values
        indicate under-estimation (pessimism).
    \item \textbf{Absolute relative error} (value accuracy):
        \[
            e_\text{abs} = \frac{1}{|\mathcal{S}_v|}
            \sum_{s\in\mathcal{S}_v} \frac{|\hat{V}(s) - V^*(s)|}{V^*(s)}.
        \]
    \item \textbf{Policy evaluation error} (primary metric, computed every 50
        episodes):
        \[
            e_\pi = \frac{1}{|\mathcal{S}_v|}
            \sum_{s\in\mathcal{S}_v} \frac{|V^{\hat{\pi}}(s) - V^*(s)|}{V^*(s)},
        \]
        where $V^{\hat{\pi}}$ is computed by model-based policy evaluation
        of the greedy (or softmax) policy induced by the current agent.
\end{enumerate}

The policy evaluation error is the primary cross-family comparison metric
because $e_\text{abs}$ has incompatible semantics across algorithm families:
for TD methods $\hat{V}(s)=\min_a Q(s,a)\approx V^*(s)$, while for
actor-critic and REINFORCE agents $\hat{V}(s)=V^{\hat{\pi}}(s)$, which
starts very high (a random policy hitting walls repeatedly produces
$V^{\hat{\pi}}\gg V^*$).

\subsection{Fairness of Comparison}

\begin{itemize}[leftmargin=*]
    \item \textbf{Same environment instances.}  Per-run, both the agent and the
        environment share the same base seed, so every algorithm faces
        identical stochastic conditions within a run.
    \item \textbf{Same number of interactions.}  All algorithms train for
        exactly 1500 episodes with at most 300 steps each; no algorithm
        receives more environment samples.
    \item \textbf{Same discount factor.}  $\gamma=0.85$ for all.
    \item \textbf{Adaptive LR for TD methods.}  The count-based schedule
        $\alpha_t = n_t^{-0.8}$ is applied uniformly to Q-learning, Double Q,
        SARSA, Expected SARSA, and G-learning, rather than hand-tuning per
        algorithm.
    \item \textbf{Fixed LR for PG/AC methods.}  REINFORCE and AC use fixed
        learning rates consistent with gradient-descent practice where
        count-based rates are less natural.
    \item \textbf{No shared state.}  Each run creates a fresh agent and
        environment instance; no gradient or Q-table information is shared
        across seeds.
\end{itemize}

\subsection{Discussion}

\paragraph{Q-learning is the best performer.}
Q-learning achieves the lowest final policy evaluation error ($e_\pi \approx
0.076$).  The key advantages are structural.  The off-policy TD target
$c + \gamma\min_{a'}Q(s',a')$ bootstraps directly toward $Q^*$ regardless
of the exploration policy.  The count-based learning rate $\alpha_t =
n_t^{-0.8}$ satisfies the Robbins--Monro conditions and adapts per
state-action pair, giving more weight to early visits while slowing once a
cell is well-characterised.  Q-learning updates every step, accumulating
up to $1500 \times 300$ individual Bellman updates, versus once per episode
for REINFORCE.

\paragraph{G-learning is the closest competitor.}
G-learning achieves $e_\pi \approx 0.094$, nearly matching Q-learning and
clearly separating from all other algorithms.  The entropy bonus encourages
the agent to maintain a spread policy early in training (when $\beta$ is
small), providing principled exploration without relying solely on
$\varepsilon$-greedy.  As $\beta$ increases linearly over 1500 episodes, the
softmin policy sharpens toward the greedy policy.  The tuned $\beta_{\min}=0.01$
(wider initial exploration than the default 0.1) accounts for the improvement
over the original configuration; the agent maps the wall-blocked corridors
before committing to a policy direction.

\paragraph{Double Q-learning shows high variance.}
Double Q corrects the minimisation bias present in standard Q-learning by
decoupling action selection from evaluation.  However, its 10-run mean
$e_\pi=0.482$ is significantly worse than Q-learning and shows the largest
standard deviation among TD methods (0.118).  The core issue is sample
efficiency: each table receives only half as many updates per episode, and
with the lower $\varepsilon_0=0.2$, state coverage is less uniform early in
training.  The signed error plots confirm that Double Q's reduced pessimism
(closer to zero $e_\pm$ early on) does not translate into faster policy
convergence on this particular environment.

\paragraph{On-policy TD methods learn more cautiously.}
SARSA's mean policy error (0.498) is substantially larger than Q-learning's
(0.076).  SARSA optimises $Q^\pi$ for the \emph{behaviour} policy
(epsilon-greedy) rather than the optimal policy: near-wall exploratory steps
incur the 1000-unit penalty, making the learned $Q^\pi$ overestimate near-wall
costs and producing an overly conservative policy.  Expected SARSA (0.266)
significantly outperforms SARSA by replacing the stochastic sample with the
analytical expectation under $\pi$, eliminating one variance source and
producing cleaner Q-table updates especially when $\varepsilon$ is large.

\paragraph{Effect of exploration.}
The epsilon-greedy scheme decays per step within each episode.  With
$\varepsilon_0=0.2$ and decay 0.9, epsilon reaches $\approx 0.001$ after 50
steps, so each episode quickly transitions to near-greedy behaviour.  The
tuning sweep confirmed that lower $\varepsilon_0$ reduces early wall-collision
damage without sacrificing coverage: since the state space is small (48 valid
states) and episodes are up to 300 steps long, even $\varepsilon_0=0.2$
provides adequate early exploration.

\paragraph{Eligibility traces hurt Actor-Critic.}
The $\lambda$ sweep revealed that $\lambda=0$ is optimal for both actor and
critic traces.  This is the opposite of what classical RL theory predicts for
long episodes.  The cause is the 1000-unit wall-collision cost: a single
off-policy exploratory step that hits a wall creates a TD error of order
$10^3$, which with $\lambda>0$ propagates backward and corrupts value and
policy parameters at states that had nothing to do with the wall.  At
$\lambda=0$, AC($\lambda$) reduces to one-step AC, and both produce identical
results ($e_\pi=0.404$) in the final comparison.  This environment-specific
finding highlights that trace benefits are contingent on the cost signal being
reasonably bounded.

\paragraph{Policy-based methods are the worst performers.}
Pure REINFORCE achieves the worst policy error (0.576).  The fundamental issue
is variance: the return $G_t = \sum_{k\geq t}\gamma^k c_k$ is a sum over the
entire remaining episode.  A single wall collision ($c \approx 1001$) dominates
$G_t$ for all preceding steps, producing a large and misleading gradient signal.
REINFORCE also does not update between steps, losing per-step Bellman
information that TD methods exploit.

Adding a baseline (REINFORCE+baseline) reduces the error to 0.536 by centering
the gradient: $\delta_t = G_t - V(s_t)$ shrinks once $V$ tracks the average
return (Figure~\ref{fig:variance_comparison}).  The tuned policy LR of 0.001
(ten times smaller than the default) is essential: at 0.01, early wall penalties
cause the policy to oscillate and never recover within 1500 episodes.

\paragraph{Hyperparameter sensitivity across families.}
TD methods with count-based learning rates are inherently robust: the rate
adapts automatically and only the exponent $\omega=0.8$ requires specification.
Policy-gradient and actor-critic methods are far more sensitive to the fixed
learning rate.  The tuning sweeps show that the optimal policy LR for REINFORCE
is 10$\times$ smaller than the commonly used default, solely because the
wall-collision penalty is three orders of magnitude larger than the per-step
movement cost.  This sensitivity is environment-specific: in environments with
bounded, moderate costs the default LR of 0.01 would likely work well.

\paragraph{Summary.}
Off-policy value-based methods—especially Q-learning—are most effective on this
tabular, cost-minimisation Gridworld because they exploit the Bellman structure
directly at every step without on-policy bias or Monte-Carlo variance.
G-learning comes close by combining TD efficiency with principled entropy-driven
exploration.  On-policy TD methods converge more slowly due to conservative
target evaluation.  Policy-gradient methods struggle because the high-variance
cost signal (wall penalties $\approx 1001$) overwhelms gradient estimates before
the policy learns to avoid walls.  The most surprising finding is that
eligibility traces ($\lambda>0$) actively hurt actor-critic performance
here---a direct consequence of the extreme wall-cost asymmetry---making
AC($\lambda$) with $\lambda=0$ equivalent to one-step AC.

LLMs (Claude, via Anthropic's Claude Code CLI) were used throughout the project
for implementation and debugging.  The LLM usage log is maintained in
\texttt{report/llm\_usage.md}; key usage sessions are summarised below.

\begin{enumerate}[leftmargin=*]
    \item \textbf{Environment porting (2026-04-02):} The \texttt{GridWorldEnv}
        class, transition matrices $P$ and $C$, and \texttt{policy\_eval} were
        ported from a reference notebook into a structured module following the
        project interface.  The only intentional deviation was replacing global
        \texttt{np.random} calls with a seeded \texttt{np.random.Generator} for
        reproducibility.  Step dynamics and matrix logic were verified to match
        the notebook exactly.

    \item \textbf{Project scaffolding (2026-04-02):} The full directory structure,
        abstract base class, stub files, configuration, and test skeletons were
        generated.  All decisions (e.g.\ count initialisation to 1, per-episode
        reset) were derived from the specification and verified against the
        reference notebook.

    \item \textbf{Algorithm implementation (2026-04-02 and 2026-04-06):}
        All nine algorithm files and their corresponding test classes
        (48 tests total) were implemented.  The implementations followed
        the mathematical specifications in the project brief; the LLM helped
        translate equations into vectorised NumPy code.  Each implementation
        was verified on a trivial 2-state 2-action MDP with known $Q^*$ before
        being run on the full Gridworld.

    \item \textbf{AC($\lambda$) bugfix and experiment runners (2026-04-23):}
        Two bugs in AC($\lambda$) were identified and fixed: (a) eligibility
        traces were only decayed at the current state, not globally, causing
        unbounded growth; (b) the actor trace used the reward-maximisation sign
        convention instead of cost-minimisation.  The LLM generated a candidate
        fix that was verified by observing policy error decreasing (from stuck
        at 1.50 to a trajectory toward 0.40) after the correction.

    \item \textbf{Hyperparameter tuning infrastructure (2026-04-26):}
        Coordinate-descent sweep cells were added to notebooks 01--04.  Each
        sweep cell uses a shared \texttt{run\_sweep()} helper that instantiates
        fresh agents over a parameter grid and records the final policy evaluation
        error.  Tuned values were consolidated into \texttt{default.yaml} and
        verified by re-running all experiments.

    \item \textbf{Report writing:} LLM used to assist in drafting, structuring,
        and refining the technical content of this report.  All theoretical
        interpretations and experimental conclusions were verified by the group
        against the observed numerical results and course material.
\end{enumerate}

The final implementation, experiment design, numerical results, and conclusions
were verified and finalised by the group.

\appendix
\section{Code Repository and Reproducibility}
\label{sec:appendix}
% ============================================================

\subsection*{Repository}

The full source code is publicly available at:

\begin{center}
    \url{https://github.com/RottenDoom/Reinforcement-Learning-Algos}
\end{center}

The repository contains all algorithm implementations, the Gridworld environment,
experiment runners, analysis notebooks, and this report source.

\subsection*{Directory Structure}

\begin{verbatim}
algorithms/          # one file per algorithm (q_learning.py, sarsa.py, ...)
environment/         # GridWorldEnv, transition matrices P and C
experiments/
    configs/
        default.yaml # single source of truth for all hyperparameters
    run_all.py       # full multi-run experiment (saves .npy results)
    run_single.py    # single-algorithm debug run
    results/         # output .npy files (git-ignored)
notebooks/           # 00–09 analysis and tuning notebooks
tests/               # test_env.py, test_algorithms.py
utils/               # metrics.py, plotting.py, schedule.py
\end{verbatim}

\subsection*{Setup}

Python 3.10 or later is required.  Clone the repository and install
dependencies into a virtual environment:

\begin{verbatim}
git clone https://github.com/RottenDoom/Reinforcement-Learning-Algos
cd Reinforcement-Learning-Algos
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
\end{verbatim}

Dependencies are \texttt{numpy}, \texttt{matplotlib}, \texttt{pyyaml},
\texttt{scipy}, and \texttt{pytest}.  No deep-learning framework is required.

\subsection*{Running the Full Experiment}

To reproduce all results from the paper, run from the project root:

\begin{verbatim}
python experiments/run_all.py
\end{verbatim}

This trains all nine algorithms for 1500 episodes across 10 seeds and writes
one \texttt{.npy} file per algorithm to \texttt{experiments/results/}.
Expected runtime is 10--20 minutes on a modern CPU.

To run a single algorithm:

\begin{verbatim}
python experiments/run_all.py --algo q_learning
\end{verbatim}

Valid algorithm names are: \texttt{q\_learning}, \texttt{double\_q\_learning},
\texttt{sarsa}, \texttt{expected\_sarsa}, \texttt{g\_learning},
\texttt{reinforce}, \texttt{reinforce\_baseline}, \texttt{actor\_critic},
\texttt{actor\_critic\_lambda}, \texttt{vanilla\_sgd}, \texttt{sgd\_momentum},
\texttt{sgd\_nesterov}, \texttt{mirror\_descent}, \texttt{natural\_pg},
\texttt{trpo}.

For a lightweight single-seed debug run with live progress output:

\begin{verbatim}
python experiments/run_single.py --algo g_learning --runs 3 --episodes 200
python experiments/run_single.py --algo actor_critic --seed 42 --verbose
\end{verbatim}

\subsection*{Reproducing Figures}

After \texttt{run\_all.py} completes, open and run
\texttt{notebooks/05\_final\_comparison.ipynb} to regenerate all figures saved
to \texttt{report/figures/}.  Individual algorithm families can be explored in
notebooks 01--04.  Hyperparameter tuning sweeps are embedded in those same
notebooks (look for cells tagged \texttt{nb0x-tune-*}).

\subsection*{Running the Tests}

The test suite covers environment correctness (48 state-transition checks) and
algorithm convergence on a minimal 2-state MDP:

\begin{verbatim}
pytest tests/ -v
\end{verbatim}

To run only environment or algorithm tests:

\begin{verbatim}
pytest tests/test_env.py -v
pytest tests/test_algorithms.py -v
\end{verbatim}

All 48 tests should pass.  The suite does not require the results files to be
present.

\subsection*{Modifying Hyperparameters}

All hyperparameters are centralised in \texttt{experiments/configs/default.yaml}.
Edit that file before re-running \texttt{run\_all.py}; no algorithm source file
needs to change.  Key parameters:

\begin{verbatim}
environment:
  gamma: 0.85          # discount factor
  n_episodes: 1500     # training episodes per run
  n_runs: 10           # independent seeds

exploration:
  epsilon_start: 0.2   # initial epsilon for epsilon-greedy
  epsilon_decay: 0.9   # per-step decay within each episode

actor_critic:
  lambda_critic: 0.0   # eligibility trace decay (0 = one-step TD)
  lambda_actor:  0.0
\end{verbatim}
